## Critical Rules Before Making Any Changes

### 1. Never commit development assets
Running `npm run dev` rewrites `snippets/vite-tag.liquid` to point to `localhost:5173`. If you commit this, the Shopify preview goes **blank white**. Always run `npm run build` and commit the production version before pushing.

**Safe commit sequence:**
```bash
npm run build
git add snippets/vite-tag.liquid assets/
git commit -m "build: your message here"
git push origin main
```

### 2. Never add files to `assets/` subdirectories
Shopify's CDN does not allow subfolders inside `assets/`. The only exception is `assets/.vite/` which is listed in `.shopifyignore` and never synced to Shopify servers.

### 3. Never edit `snippets/vite-tag.liquid` manually
This file is auto-generated by `vite-plugin-shopify` on every build. Any manual edits will be overwritten on the next `npm run build`.

### 4. Don't add Liquid variables to React components
React fetches all data via Shopify's JSON APIs at runtime. Do not try to pass Liquid variables (e.g. `{{ product.title }}`) into React components — they live in separate execution contexts.

## Common Tasks

### Add a new page / route
1. Create a new component in `frontend/components/`
2. Add a `<Route path="/your-path" element={<YourComponent />} />` in `App.jsx`
3. No Liquid changes needed — React Router handles all navigation

### Add a new product field
1. Update `fetchProducts()` or `fetchProduct()` in `frontend/utils/shopify.js`
2. Add the new field to the returned object
3. Use it in the relevant component

### Change a colour or spacing value
1. Edit the CSS variable in `frontend/index.css` under `:root`
2. Do NOT hardcode the value in component files

### Add a new component
1. Create `frontend/components/MyComponent.jsx` and `MyComponent.css`
2. Import the CSS file in the JSX file
3. Use existing CSS variables for all colours and spacing
4. Follow BEM naming: `.my-component__element--modifier`

### Deploy changes
```bash
npm run build
git add -A
git commit -m "feat: your description"
git push origin main
```
Shopify picks up the changes automatically via GitHub integration.
